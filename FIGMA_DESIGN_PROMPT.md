# Figma Design Prompt for Traffic Management System

## Project Name
Traffic Management System for Energy and Cost Efficient Route Optimization

## Design Goal
Create a modern web dashboard style interface for a traffic management and route optimization system. The system helps users select the most energy-efficient and cost-effective route, not only the fastest or shortest route.

The interface should feel professional, intelligent, clean, and research-oriented. It should look like a smart transportation analytics platform that combines live route data, energy estimates, traffic conditions, and route recommendation results.

## Target Users
- Drivers looking for efficient routes
- Researchers evaluating route performance
- Students and panel members reviewing the prototype

## Core Design Direction
- Clean and modern dashboard interface
- Web application layout, desktop first but responsive for mobile
- Professional transportation or smart-city visual style
- Emphasis on route comparison, metrics, and recommendation clarity
- Use maps, cards, charts, and route status indicators

## Suggested Visual Style
- Primary color: Deep blue or teal
- Secondary color: Green for efficiency and savings
- Accent color: Orange or red for traffic congestion warnings
- Background: Light neutral with subtle map/grid texture
- Typography: Clean and readable sans-serif
- Mood: Smart mobility, sustainability, and data-driven decision making

## Main Screens to Design

### 1. Landing Page
Purpose:
Introduce the system and explain its value.

Must include:
- Hero section with title and subtitle
- Short explanation of energy-efficient route optimization
- Call-to-action button such as Start Route Analysis
- Visual area showing map preview, route lines, and traffic indicators
- Section showing system features
- Section explaining how the system works

### 2. Route Input Page / Main Dashboard
Purpose:
Allow user to enter trip details and request route analysis.

Must include:
- Origin input field
- Destination input field
- Optional vehicle type selector
- Optional fuel type selector
- Optional fuel price input
- Button for Analyze Routes
- Embedded map area
- Sidebar or panel for trip filters and route settings

### 3. Route Comparison Results Page
Purpose:
Show multiple route options and compare them.

Must include:
- 2 to 3 route cards
- Each route card should display:
  - Route name or label
  - Distance
  - Estimated travel time
  - Traffic level
  - Estimated fuel or energy consumption
  - Estimated travel cost
  - Sustainability rating or efficiency score
- Highlight one card as Recommended Route
- Comparison table or visual chart for all route metrics
- Map with different route colors

### 4. Recommendation Detail Panel
Purpose:
Explain why a route is recommended.

Must include:
- Recommendation badge
- Score breakdown
- Main reasons for recommendation
- Savings summary such as time saved, fuel saved, or cost reduced
- Risk or limitation note if traffic is changing

### 5. Research Analytics / Admin View
Purpose:
Support thesis demonstration and route evaluation.

Must include:
- Table of tested routes
- Summary cards for average time, average fuel use, average cost
- Chart comparing recommended route versus fastest and shortest route
- Export or report style layout for academic presentation

### 6. Mobile Responsive View
Purpose:
Show how the design adapts to smaller screens.

Must include:
- Simplified route input form
- Route recommendation card
- Mini map
- Swipeable route comparison cards

## Important Components
- Navigation bar
- Search and location input fields
- Route cards
- Statistic cards
- Recommendation banner
- Traffic status badges
- Energy or fuel consumption indicator
- Cost estimation widget
- Map container
- Comparison chart section
- Footer or project info section

## Recommended User Flow
1. User opens landing page
2. User clicks Start Route Analysis
3. User enters origin and destination
4. System displays route options
5. User reviews route comparison
6. System highlights recommended route
7. User sees explanation and efficiency metrics
8. Research/admin page shows evaluation summary

## Layout Guidance
- Use a left panel for filters and inputs
- Use a large central map area
- Use route cards on the right or below the map
- Use top summary cards for key metrics
- Keep the recommended route visually dominant

## Design Keywords for Figma AI
smart mobility, traffic dashboard, route optimization, sustainable transportation, energy efficient navigation, modern analytics interface, map-based dashboard, intelligent travel assistant, professional research prototype, clean transport UI

## Final Instruction for the Designer or Figma AI
Design a polished website UI for a Traffic Management System that recommends energy-efficient and cost-effective travel routes. The design should include a landing page, route input dashboard, route comparison page, recommendation panel, analytics page, and responsive mobile view. Focus on map-based navigation, route metrics, traffic indicators, fuel or energy consumption estimates, and cost savings visualization. The recommended route must stand out clearly and the interface should feel professional, modern, and suitable for a thesis prototype.
